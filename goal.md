# Цель проекта и перечень артефактов
## 1. Формулировка бизнес-цели проекта

**Цель проекта** — разработать систему прогнозирования ежедневных продаж для сети магазинов на основе исторических данных.
Модель должна **предсказывать объём продаж по каждому магазину и категории товаров**, учитывая временные факторы, праздники, транзакционную активность и внешние экономические показатели (например, цену нефти).

Полученный инструмент позволит:

- улучшить планирование запасов и закупок;

- уменьшить издержки, связанные с избыточными остатками;

- повысить точность планирования персонала и логистики;

- заранее оценивать влияние праздников и промо-активностей.

Таким образом, задача объединяет **аналитическую часть** (исследование данных, выявление закономерностей) и **ML-часть** (разработка модели прогнозирования).

## 2. Формулировка технической цели

Создать полный аналитико-модельный pipeline, включающий:

- загрузку данных в SQL-базу;

- предобработку данных (cleaning, feature engineering);

- исследовательский анализ данных (EDA);

- формулировку и проверку гипотез;

- построение baseline и продвинутой ML-модели;

- анализ качества, визуализации и интерпретацию результатов;

- создание мини-дашборда для отображения ключевых метрик и прогнозов.

Финальный результат должен быть воспроизводим и пригоден для оценки компетенций в области аналитики данных и машинного обучения.

## 3. Целевая метрика качества

Основные метрики для оценки точности модели:

- **RMSE (Root Mean Squared Error)** — основной показатель.

- **MAPE (Mean Absolute Percentage Error)** — дополнительный показатель для интерпретации относительной ошибки.

Модель сравнивается с несколькими baseline-подходами (простая сезонность, лаги), чтобы показать реальный прирост качества.

## 4. Список артефактов проекта

### 4.1. SQL-артефакты

- `create_tables.sql` — скрипт создания таблиц.

- `load_data.sql` — скрипт загрузки CSV в SQL.

- SQL-запросы для проверки качества данных и аналитических выборок.

### 4.2. Ноутбуки

- `data_exploration.ipynb` — первичная проверка структуры данных.

- `eda.ipynb` — полноценный EDA с графиками и гипотезами.

- `feature_engineering_and_baseline_models.ipynb` — создание признаков (time-based, lag, rolling) и простые модели.

- `models_and_error_analysis.ipynb` — ансамблевые модели и сравнение результатов и анализ ошибок.

### 4.3. Excel-отчёты

- `eda_pivot_tables.xlsx` — сводные таблицы и графики (категории/месяцы/магазины).

### 4.4. Дашборд

- папка `dashboard/` со Streamlit или Plotly Dash приложением:

  - визуализация временных рядов,

  - прогноз по выбранному магазину,

  - ключевые показатели.

### 4.5. Готовый отчёт

- `final_report.pdf` или `.md`:

  - цель,

  - описание данных,

  - EDA и гипотезы,

  - описание модели,

  - анализ ошибок,

  - выводы и рекомендации.
